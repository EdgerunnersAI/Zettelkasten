**Goal:** Land the "typed core + JSONB extension" pattern on `content.canonical_zettels`, `content.canonical_chunks`, `kg.kg_nodes`, `kg.kg_edges`. Single migration, all-additive nullable-default columns. Buys "any-type-of-Zettel without DDL" forever.

**Files:**

- Modify: `tests/unit/supabase_v2/test_schema_files.py` (allowlist)

**Rationale + citations (from R6 research):**
- [Crunchy Data — Indexing JSONB in Postgres](https://www.crunchydata.com/blog/indexing-jsonb-in-postgres) — `jsonb_path_ops` ~3× smaller than default `jsonb_ops` for `@>` containment.
- [pganalyze — Postgres GIN Indexes](https://pganalyze.com/blog/gin-index) — partial GIN per kind, not blanket.
- [Notion — The data model behind Notion's flexibility](https://www.notion.com/blog/data-model-behind-notion) — `kind` discriminator + payload, schema_ver per row.
- [Neo4j 2025 dynamic labels and types](https://github.com/neo4j/neo4j/wiki/Neo4j-2025-changelog) — `entity_type` as text not enum; runtime-resolved.

**Steps:**

- [ ] **Step 1: Write the migration**
  ```sql

  -- Phase 8.5.R6 — additive forward-compat columns. All NULLABLE DEFAULT, RLS-unchanged.
  
  ALTER TABLE content.canonical_zettels
    ADD COLUMN IF NOT EXISTS kind        text     NOT NULL DEFAULT 'zettel',
    ADD COLUMN IF NOT EXISTS schema_ver  smallint NOT NULL DEFAULT 1,
    ADD COLUMN IF NOT EXISTS attrs       jsonb    NOT NULL DEFAULT '{}'::jsonb;
  
  ALTER TABLE content.canonical_chunks
    ADD COLUMN IF NOT EXISTS modality       text     NOT NULL DEFAULT 'text',
    ADD COLUMN IF NOT EXISTS schema_ver     smallint NOT NULL DEFAULT 1,

  ALTER TABLE kg.kg_nodes ADD COLUMN IF NOT EXISTS attrs jsonb NOT NULL DEFAULT '{}'::jsonb;

  CREATE INDEX IF NOT EXISTS cz_kind_source_idx
    ON content.canonical_zettels (kind, source_type);

    ON content.canonical_zettels USING gin (attrs jsonb_path_ops);

    ON content.canonical_chunks USING gin (embedding_meta jsonb_path_ops);

    ON kg.kg_nodes USING gin (attrs jsonb_path_ops);
  ```

- [ ] **Step 1: Write the introspection RPC**
  ```sql
  -- supabase/website/_v2/33_introspect_auth_users_dependents.sql
  CREATE OR REPLACE FUNCTION core.introspect_auth_users_dependents()
  RETURNS TABLE(schema_name text, table_name text, fk_column text)
  LANGUAGE sql STABLE SECURITY DEFINER SET search_path = public AS $$
    WITH RECURSIVE deps AS (
      SELECT kcu.table_schema::text AS schema_name,
             kcu.table_name::text   AS table_name,
             kcu.column_name::text  AS fk_column,
             ccu.table_schema::text AS ref_schema,
             ccu.table_name::text   AS ref_table
        FROM information_schema.key_column_usage kcu
        JOIN information_schema.referential_constraints rc
          ON kcu.constraint_name = rc.constraint_name
         AND kcu.constraint_schema = rc.constraint_schema
        JOIN information_schema.constraint_column_usage ccu
          ON ccu.constraint_name = rc.unique_constraint_name
         AND ccu.constraint_schema = rc.unique_constraint_schema
       WHERE ccu.table_schema = 'auth' AND ccu.table_name = 'users'
      UNION
      SELECT kcu.table_schema::text, kcu.table_name::text, kcu.column_name::text,
             ccu.table_schema::text, ccu.table_name::text
        FROM information_schema.key_column_usage kcu
        JOIN information_schema.referential_constraints rc
          ON kcu.constraint_name = rc.constraint_name
         AND kcu.constraint_schema = rc.constraint_schema
        JOIN information_schema.constraint_column_usage ccu
          ON ccu.constraint_name = rc.unique_constraint_name
         AND ccu.constraint_schema = rc.unique_constraint_schema
        JOIN deps d ON d.schema_name = ccu.table_schema AND d.table_name = ccu.table_name
    )
    SELECT DISTINCT schema_name, table_name, fk_column FROM deps;
  $$;
  
  REVOKE ALL ON FUNCTION core.introspect_auth_users_dependents() FROM PUBLIC;
  GRANT EXECUTE ON FUNCTION core.introspect_auth_users_dependents() TO service_role;
  
  NOTIFY pgrst, 'reload schema';
  ```

- [ ] **Step 2: Write production purge helper**
  ```python

  """Pre-delete cleanup of CASCADE blockers before auth.admin.delete_user.
  
  Used both by the production "delete account" flow and by
  tests/integration/v2/test_user_cascade.py.
  """
  from __future__ import annotations
  import uuid
  from pydantic import BaseModel
  from website.core.supabase_v2.client import get_v2_client
  
  class UserPurgeReport(BaseModel):
      storage_objects_deleted: int = 0
      kasten_members_removed: int = 0
      pricing_subscriptions_cancelled: int = 0
      chat_sessions_deleted: int = 0
  
  def purge_user_dependencies(profile_id: uuid.UUID) -> UserPurgeReport:
      """Pre-flight cleanup of FK blockers; safe to call before auth.admin.delete_user."""
      client = get_v2_client()
      report = UserPurgeReport()
      # storage.objects.objects_owner_fkey
      objs = client.schema("storage").table("objects").delete().eq("owner", str(profile_id)).execute()
      report.storage_objects_deleted = len(objs.data or [])
      # kasten_members
      km = client.schema("rag").table("kasten_members").delete().eq("member_profile_id", str(profile_id)).execute()
  ```

  ```python
  import ast, pathlib, pytest

  BANNED_ATTRS = {"node_id"}
  BANNED_CALLS = {"candidate_to_legacy_dict"}
  ALLOWLIST = {  # files permitted to define/expose the alias itself

      tree = ast.parse(path.read_text(encoding="utf-8"))
      for node in ast.walk(tree):

              yield path, node.lineno, f"node.{node.attr}"
          if isinstance(node, ast.Call) and isinstance(node.func, ast.Name) and node.func.id in BANNED_CALLS:
              yield path, node.lineno, node.func.id
  
  def test_acl_001_no_legacy_alias_consumers():
      bad = []

          bad.extend(_violations(py))
  ```
